## Quality Assessment

### Data Quality Metrics

- **Zero exact duplicates** across all 521 pairs
- **Zero near-duplicates** (matching first 80 characters)
- **No very short answers** -- minimum answer length is 1,023 chars
- Answer length: mean 1,763 chars, median 1,746 chars, max 3,015 chars
- System prompt length: mean 2,118 chars (includes code snippets)

### DeepSeek V4 Quality (287 pairs)

DeepSeek V4-generated pairs are **comparable quality** to hand-curated pairs:

| Source | Avg Answer Length | Avg Code Refs/Answer | Min/Max Length |
|--------|-------------------|---------------------|----------------|
| bug-qa (hand) | 1,805 chars | 4.0 | 1,401 / 2,467 |
| crossfile-qa (hand) | 2,048 chars | 8.7 | 1,494 / 2,640 |
| deepseek-v4 (gen) | 1,664 chars | 6.3 | 1,023 / 2,624 |

- **5 pairs** (1.7%) reference file names not present in their system prompt (minor hallucinations):
  - `App/FlavorConfig.swift` (correct name is `Domain/FlavorConfig.swift`)
  - `Services/CrashReporting/RemoteCrashReporter.swift` (does not exist)
  - 3 others with path normalization issues (e.g., `LicenseStore.swift` without package prefix)
- **Zero** answers contain generic filler phrases
- DeepSeek answers are slightly shorter on average but have more code references than bug-qa

### System Prompt Code Freshness

Spot-checked 3 high-traffic files against current codebase:
- `AudioRecorder.swift` -- system prompt matches current code (imports, class declaration, properties)
- `EngineRouter.swift` -- matches
- `AppEnvironment.swift` -- matches

However, the actual `AudioRecorder.swift` now has properties (`bufferFlowMonitorTask`, `recordingDiedContinuation`, `recordingDiedStream`) not present in system prompts. System prompts contain **partial file excerpts**, not full files -- this is expected but means the model won't learn about newer additions.

### Factual Accuracy (Spot-Check of 15 Pairs)

Checked pairs across audio, concurrency, output, licensing subsystems:
- **All 15 verified accurate** -- answers correctly reference actual patterns, constants, and architectural decisions in the codebase
- Bug scenarios reference real issue numbers and actual code patterns
- Concurrency answers correctly explain MainActor deadlock patterns, `try? await Task.sleep()` hazards
- Output answers correctly describe CGKeyEventSynthesizer behavior

### Generation Strategy Notes

1. **System prompts must include current code** -- extract fresh snippets from HEAD, not cached versions
2. **DeepSeek V4 quality is sufficient** -- 98.3% clean rate, comparable to hand-curated. Continue using for bulk generation
3. **Watch for file path hallucinations** -- validate all `source_files` references against actual codebase
4. **Preserve metadata in raw files** -- `id`, `category`, `difficulty`, `subsystem`, `source_files` are valuable for future audits
5. **TunedVoiceKit needs its own system prompt prefix** -- current prompt says "macOS app" but Kit is a shared package used by watchOS too
6. **Backend pairs need TypeScript system prompt** -- current prompt template is Swift-focused; need a separate template for edge functions
