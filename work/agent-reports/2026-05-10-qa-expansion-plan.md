# Q&A Expansion Plan and Generation Script

Date: 2026-05-10
Task: Find original DeepSeek V4 generation script, plan expansion, write new script

## What was done

### 1. Found original generation scripts

Located at `work/qa-pairs/` (NOT in the qwen-rft-pipeline repo):
- `generate_qa.py` -- single-file Q&A generation (produced the 287 deepseek-v4 pairs)
- `generate_crossfile_qa.py` -- cross-file Q&A generation (produced 47/67 crossfile pairs)
- `generate_bug_qa.py` -- bug-grounded Q&A (produced 159 bug-qa pairs)
- `gen_pair.py` -- simple single-pair writer (helper utility)
- `METHODOLOGY.md` -- detailed methodology documentation

Key finding: The original scripts used `claude -p --model sonnet` (Claude subscription CLI) NOT the DeepSeek API. The file was named `deepseek-v4-qa.jsonl` because early attempts used DeepSeek V4 via OpenRouter but that was abandoned due to reliability issues (connection resets, 29-min stalls). The actual 287 pairs were generated by Claude Sonnet.

### 2. Analyzed existing data format

Schema per line in JSONL:
```json
{
  "id": "<prefix>_<NNN>",
  "category": "explain|implement|debug|architecture|refactor",
  "difficulty": "L1|L2|L3",
  "subsystem": "<subsystem>",
  "source_files": ["relative/path.swift"],
  "messages": [
    {"role": "system", "content": "<persona + code excerpt>"},
    {"role": "user", "content": "<question>"},
    {"role": "assistant", "content": "<answer>"}
  ]
}
```

Distribution: 287 pairs, 15 subsystems, 5 categories, 3 difficulty levels. All single-file (source_files always length 1). System prompts include ~2000 chars of code context.

### 5. Wrote generation script

`scripts/generate_qa_pairs.py` -- configurable, resumable, idempotent:
- Supports DeepSeek direct API and OpenRouter as fallback
- Supports 3 source types: app (Swift), kit (Swift), backend (TypeScript)
- Has `--dry-run` mode
- Resume support via prefix-based deduplication (same pattern as original)
- JSON schema validation before writing
- Cross-file slice support for TunedVoiceKit
- Updated prompt templates with more implementation and code_navigation emphasis
- Uses stdlib only (urllib, no pip dependencies)
- Syntax verified

## Key findings

1. **The "DeepSeek V4" pairs were actually generated by Claude Sonnet.** The name is misleading. DeepSeek via OpenRouter was tried first but abandoned due to China infrastructure routing issues.

2. **Original scripts live in `work/qa-pairs/`, not in any repo.** They were standalone one-off scripts, not committed to qwen-rft-pipeline.

3. **The data exists in two locations**: `work/qa-pairs/*.jsonl` (originals) and `repos/qwen-rft-pipeline/data/raw/*.jsonl` + `repos/llm-toolkit/training-data/tunedvoice-rft/raw/*.jsonl` (copies).

4. **The original pipeline had 3 passes**: single-file (287), cross-file (47-67), bug-grounded (131-159). The new script unifies all three patterns.

## Learnings

1. **New facts**: DeepSeek V4 via OpenRouter routing to China infrastructure caused 29-minute stalls and connection resets during the original generation. Claude Sonnet via `claude -p` had zero failures.

2. **Reference doc updates**: The qa-audit-and-gaps.md says "Generated by DeepSeek V4" but should note they were generated by Claude Sonnet (after DeepSeek was abandoned).

3. **Agent prompt corrections**: None needed.
