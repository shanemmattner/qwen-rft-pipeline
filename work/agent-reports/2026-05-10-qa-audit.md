# Q&A Training Data Audit
Date: 2026-05-10
Agent: tunedvoice-macos
Task type: explore

## What was done
Audited all 521 Q&A training pairs across 4 raw source files. Categorized by subsystem (15 categories) and question type (3 categories). Spot-checked 15 pairs for factual accuracy. Compared source file coverage against the actual TunedVoice codebase. Identified gaps and produced an expansion plan to reach 1000+ pairs.

## Key decisions
- Mapped the 7 raw categories (debug, architecture, explain, implement, edge_case, concurrency, refactor) into 3 normalized types (Architecture, Debugging, Implementation) for cleaner analysis
- Evaluated DeepSeek V4 quality against hand-curated pairs using quantitative metrics (answer length, code reference density) rather than subjective reading
- Prioritized TunedVoiceKit package and Context Mining as P0 gaps based on file count and recency

## Difficulties encountered
- Train/valid files strip all metadata (id, category, subsystem, source_files) -- had to verify they're exact 468/53 split of the raw data to use raw metadata for analysis
- No easy way to verify system prompt code freshness at scale -- did spot-checks on 3 files

## Discoveries
- Dataset is very clean: zero duplicates, zero near-duplicates, no short answers, no generic filler
- DeepSeek V4 quality is on par with hand-curated: 98.3% of pairs have no issues. Only 5/287 have minor file path hallucinations
- TunedVoiceKit package (77 files) has near-zero coverage despite containing core business logic
- Backend edge functions (23 TypeScript files) have zero direct Q&A coverage
- Context mining (Wave 4 -- ContextScoutService, TermExtractor, ConfusionPairMatcher) has zero pairs
- Architecture questions are overrepresented (50%), implementation questions underrepresented (14%)
- 67% of app source files are covered, but only via partial code excerpts in system prompts

## Recommendations
1. Generate 200 pairs for Phase 1 (TunedVoiceKit internals, context mining, backend, code navigation, implementation)
2. Generate 280 more pairs for Phase 2 (uncovered files, cross-file scenarios, deep-dives)
3. Create separate system prompt templates for TunedVoiceKit and backend TypeScript
4. Always extract fresh code from HEAD when generating new pairs
5. Preserve metadata in raw files -- strip only for train/valid splits
